# PHP 实习面试速记大纲

## 一、PHP 基础常问（Q&A 简短版）

- **1. PHP 常见超全局变量？**  
  `$_GET`、`$_POST`、`$_REQUEST`、`$_SERVER`、`$_SESSION`、`$_COOKIE`、`$_FILES`、`$_ENV`

- **2. echo 和 print 区别？**  
  `echo`：无返回值，可输出多个参数，稍快；`print`：有返回值1，只接收一个参数。

- **3. == 和 === 区别？**  
  `==`：只比较值，会类型转换；`===`：比较值 + 类型，更严格，一般推荐。

- **4. include 和 require 区别？**  
  `include` 失败：warning，脚本继续；`require` 失败：fatal error，脚本中断。  
  `include_once` / `require_once` 防重复引入。

- **5. PHP 常见数据类型？**  
  标量：int, float, string, bool  
  复合：array, object  
  特殊：null, resource

- **6. 什么是关联数组？**  
  下标为字符串的数组，类似 map/dictionary。

## 二、Web / HTTP 基础

- **7. GET 和 POST 区别？**  
  GET：参数在 URL，长度有限，多用于获取数据；  
  POST：参数在请求体，长度相对不限，多用于提交表单/数据。

- **8. HTTP 请求大致流程？**  
  输入 URL → DNS 解析 → 建立 TCP 连接 → 浏览器发 HTTP 请求 → 服务器（PHP+DB）处理 → 返回响应 → 浏览器渲染。

- **9. 常见 HTTP 状态码？**  
  200 成功；301/302 重定向；400 请求错误；401 未授权；403 禁止；404 未找到；500 服务器错误。

- **10. session 与 cookie 区别？**  
  cookie：存浏览器端，大小有限，不适合敏感信息；  
  session：存服务器端，通过 cookie 里的 session_id 关联，更适合存敏感数据。

## 三、数据库 / MySQL

- **11. 常见 SQL 操作？**  
  `SELECT`、`INSERT`、`UPDATE`、`DELETE`，加上 `WHERE` 条件、`ORDER BY` 排序、`LIMIT` 分页。

- **12. 主键与自增 ID？**  
  主键：唯一标识一行，不重复、不为 NULL；  
  自增 ID：主键的一种，插入时自动递增。

- **13. 什么是索引？作用？**  
  索引像目录/索引，加快查询速度；  
  缺点：占空间，写入/更新会慢一点；常给查询条件字段建索引。

- **14. 如何防止 SQL 注入？**  
  使用预处理 / 参数化查询（PDO prepared statement），不要直接拼接用户输入到 SQL 中。

## 四、PHP 面向对象（OOP）

- **15. 类和对象？**  
  类是模板/蓝图，对象是根据类创建出的具体实例。

- **16. public / protected / private 区别？**  
  public：外部、子类都可访问；  
  protected：本类和子类可访问；  
  private：仅本类内部可访问。

- **17. 什么是继承？好处？**  
  子类自动拥有父类属性和方法；  
  好处：代码复用、抽象公共逻辑、便于扩展。

- **18. 什么是多态（简单说）？**  
  同一方法名在不同类中实现不同行为，比如都叫 `send()`，但一个发邮件、一个发短信。

## 五、框架 & MVC（如被问 Laravel / ThinkPHP）

- **19. 用过哪些 PHP 框架？**  
  可回答：Laravel / ThinkPHP / 原生 PHP（视自己情况）。

- **20. MVC 分层？**  
  M（Model）：数据、数据库逻辑  
  V（View）：页面展示  
  C（Controller）：接收请求，调用模型，选择视图

- **21. 一次请求的大致流程（框架中）？**  
  路由匹配 → 控制器方法 → 调用模型/服务 → 返回视图或 JSON。

## 六、常见场景题

- **22. 简单描述实现“用户登录功能”的步骤？**  
  前端表单提交账号密码 → 后端接收参数 → 数据库根据用户名查记录 → `password_verify` 校验密码 → 成功后写入 session（记录用户 id/用户名）→ 返回结果/跳转。

- **23. 文件上传要注意哪些问题？**  
  限制文件类型/大小；不要直接用用户文件名当最终文件名；  
  上传到固定目录；必要时校验后缀与 MIME 类型；避免上传可执行脚本。

- **24. 页面很慢如何排查？**  
  先看前端还是后端慢（浏览器 Network）；  
  后端看数据库是否有慢查询/缺索引；  
  加日志或 var_dump 定位卡住的位置。

## 七、简单算法/函数常见点

- **25. 找数组最大/最小值？**  
  可以手写遍历比较，也可用 `max($arr)` / `min($arr)`。

- **26. 去掉字符串两端空格函数？**  
  `trim()`；左侧：`ltrim()`，右侧：`rtrim()`。

- **27. 判断变量是否存在/为空？**  
  `isset($var)`：是否已定义且不为 null；  
  `empty($var)`：是否为空（含 0、'0'、''、null、false、[]）。

## 九、项目经历：PHP 简易博客管理系统（重点可讲）

> **项目名称**：基于 PHP 的简易博客管理系统（个人项目）  
> **技术栈**：PHP 7、MySQL、Apache/Nginx、HTML/CSS（可加 Bootstrap）

### 1. 功能概览

- 用户注册、登录、退出（基于 `session` 的登录状态管理）
- 文章的新增、编辑、删除、列表展示与分页、详情页
- （可选）文章评论功能：登录用户可对文章评论

### 2. 主要模块

- **用户模块**
  - 注册：检查用户名重复，密码用 `password_hash` 加密存储
  - 登录：使用 `password_verify` 校验密码
  - 登录状态：`$_SESSION` 记录用户 id/用户名，退出时销毁 session

- **文章模块**
  - 新增/编辑/删除文章：仅登录用户可操作
  - 列表分页：按时间倒序显示，使用 `LIMIT offset, page_size`
  - 详情页：根据文章 id 查询并显示全文

- **权限控制**
  - 未登录用户：只能浏览文章和评论
  - 登录用户：可发文/评论
  - 删除/编辑限制为作者本人（根据文章的 `user_id` 判断）

### 3. 数据库设计（简述）

- `users`：`id`（主键自增）、`username`、`email`、`password`、`created_at`
- `posts`：`id`、`user_id`、`title`、`content`、`created_at`、`updated_at`
- `comments`（可选）：`id`、`post_id`、`user_id`、`content`、`created_at`  
  为常用查询字段（如 `posts.user_id`、`comments.post_id`）建立索引。

### 4. 安全与规范

- 使用 PDO 预处理（prepared statements）防止 SQL 注入  
- 输出用户内容时使用 `htmlspecialchars`，减少 XSS 风险  
- 封装公共初始化文件（如 `init.php`），统一 `session_start()` 和数据库连接

### 5. 收获总结（面试用话术）

- 熟悉了从表单提交 → PHP 接收 → 操作 MySQL → 页面展示的完整流程  
- 对 MVC 分层有了初步实践：尝试分离视图、业务逻辑和数据库操作  
- 遇到 bug 时通过报错信息 + 日志/var_dump 排查，再结合文档和搜索解决问题
