```

Prometheus → Grafana → Alertmanager

【分级告警】(必背!)

【优化案例】
问题: 告警太多,造成告警疲劳
解决: 告警聚合 + 告警抑制 + 调整阈值
结果: 告警数量减少70%,准确率提高
```

### 王牌3: Harbor问题解决(体现学习能力!)

```
【问题】
Harbor容器启动失败,报错"failed to initialize logging driver"

【分析过程】(必背!)
1. 查看容器状态: docker-compose ps
2. 查看错误日志: docker logs
3. 发现无法连接到端口1514
4. 查看docker-compose.yml
5. 理解问题: harbor-log是日志中心,其他容器启动时它还未就绪

【解决方案】
1. 调整docker-compose.yml启动顺序
2. 先启动harbor-log容器
3. 等待10秒确保日志服务就绪
4. 再启动其他容器

【收获】
- 理解了Docker容器的依赖关系
- 学会了通过日志分析问题
- 学会了查GitHub issues解决问题
```

## 📚 核心技术知识点(必背!)

### 1. Docker核心知识

```
【Dockerfile最佳实践】
- 使用小的基础镜像(alpine)
- 多阶段构建优化镜像大小
- 合并RUN命令减少层数
- 使用.dockerignore排除文件
- 非root用户运行

【docker-compose】
- 服务定义
- 网络配置
- 卷挂载
- 依赖关系(depends_on)
- 启动顺序控制
```

### 2. Kubernetes核心概念

```
【核心资源】
- Pod: 最小部署单位
- Deployment: 管理Pod副本
- Service: 服务发现和负载均衡
- Ingress: 外部访问
- ConfigMap/Secret: 配置管理

【滚动更新】
- Deployment的滚动更新策略
- maxSurge和maxUnavailable
- 回滚机制(kubectl rollout undo)
- 健康检查(liveness/readiness probe)
```

### 3. 可观测性三大支柱

```
【日志】
- 记录事件
- 集中式日志收集
- 日志轮转

【指标】
- 量化数据
- Prometheus采集
- Grafana可视化

【链路追踪】
- 请求调用链
- OpenTelemetry(理论)
```

### 4. CI/CD核心概念

```
【灰度发布】(必背!)
- 滚动更新: 逐步替换旧版本
- 蓝绿部署: 两套环境快速切换
- 金丝雀发布: 先发布到小部分用户
- A/B测试: 不同版本对比测试

【特性开关】
- 通过配置控制功能开关
- 不需要重新部署
- 支持A/B测试
- 工具: Unleash/Flagsmith

【回滚机制】
- 保留上一个版本
- 发现问题快速回滚
- 自动回滚机制
```

### 5. 后端开发基础(补充理论!)

```
【RESTful API】(必背!)
- HTTP方法: GET(查询)/POST(创建)/PUT(更新)/DELETE(删除)
- 状态码: 2xx成功/3xx重定向/4xx客户端错误/5xx服务端错误
- 幂等性: GET/PUT/DELETE是幂等的,POST不是
- 资源设计: /users, /users/{id}

【PostgreSQL基础】
- ACID特性: 原子性/一致性/隔离性/持久性
- 事务隔离级别: 读未提交/读已提交/可重复读/串行化
- 索引: B-Tree索引,加速查询
- EXPLAIN: 查看查询执行计划

【Redis基础】
- 5种数据结构: String/Hash/List/Set/ZSet
- 缓存策略: LRU/LFU
- 过期时间: TTL
- 持久化: RDB/AOF
- 消息队列: List的LPUSH/RPOP

【认证授权】(必背!)
- JWT: Header.Payload.Signature,无状态
- OAuth2: 授权码模式/客户端模式/密码模式/隐式模式
- RBAC: 用户-角色-权限
- Session vs Token: Session有状态,Token无状态
```
